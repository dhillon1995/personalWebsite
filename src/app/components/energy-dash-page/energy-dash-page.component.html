Energy Dashboard UI
This project was to prove the concept of a company’s industrial energy monitoring device. This device attaches at the fuse box of the location you would like to monitor, and it would send data about how much energy you are using and what device is using it, via LoRa to a WIFI connected hub or gateway. This allowed locations to see exactly what’s using the most amount of energy near real time and over a longer period of time. The home page was pretty simple, I made 6 cards to simply display the ‘highlight’ information that the customers were most interested in. The most useful cards are on the top row and the secondary info is on the second row of the home page, with historical and specific data easily viewable by other tabs or pages. The other tabs were there for specific and historical data. These pages were simply designed to display data that the user wants to see, they can specify this by using time scope and data type filters. This allows sites to compare how energy is used during different seasons throughout the year. One of the challenges with this project was working with the very strange back end that went along with this project. The company had attempted to get this product going various times throughout the years, and this resulted in a peculiar PHP script that ran periodically on the server, pulling data and then placing it into a JSON file for you to use, instead of a more conventional request and response API.
